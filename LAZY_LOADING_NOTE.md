# Lazy Loading 범위 최적화 노트

현재 index.html은 카드 이미지에만 `loading="lazy"`를 적용합니다.

## 왜 카드만 lazy인가?
- 상단 로고/메인 CTA는 첫 화면(Above the fold)이므로 즉시 보여야 전환율이 올라갑니다.
- 카드 이미지는 아래로 스크롤해서 보는 경우가 많아 lazy가 효과적입니다.

## 더 최적화하고 싶다면
- 첫 줄(모바일 2개/PC 4개)만 eager 로딩, 나머지는 lazy로 분리할 수 있습니다.
- 단, 파일 관리가 복잡해질 수 있어 “지금 당장 효과 + 안정성” 관점에서 카드 전체 lazy가 무난합니다.
